Adds a `#![no_std]` crate that generates bindgen bindings from the standard C headers (`zkvm_accelerators.h`, `zkvm_io.h`) for `riscv64im-unknown-none-elf`, keeping the headers in `standards/` canonical (no duplication).

- [Marcin] As part of the [Rust bindings PR](https://github.com/eth-act/zkvm-standards/pull/30), added the missing `standards/io-interface/zkvm_io.h` header to the standards repo.

### M4 — Standardize assumptions about the ELF and zkVM processing

- [Marcin] ["Static Library and Linker Script" proposal](https://github.com/eth-act/zkvm-standards/pull/28) - new standard, ongoing discussion
- [Marcin] ZisK conformance toward the `no_std` static-library guest model required by the standards:
    - [Made `ziskos` `no_std`-compatible for the ZisK target](https://github.com/0xPolygonHermez/zisk/pull/965), removing transitive `std` from the static archive.
    - [Removed `std` from `libziskos.a`](https://github.com/eth-act/zisk/pull/1) with a CI check that fails the build if any `std` object files are bundled.
    - [Split `libziskos.a` into a dedicated `ziskos-staticlib` crate](https://github.com/0xPolygonHermez/zisk/pull/984) with CI verification of the required exported symbols (`_start`, `read_input`, `write_output`, `zkvm_init`/`zkvm_deinit`, all 18 `zkvm_*` accelerators).
    - Build/config fixes: [default-features handling](https://github.com/0xPolygonHermez/zisk/pull/1005), [`zisk_guest` cfg flag for conditional compilation](https://github.com/0xPolygonHermez/zisk/pull/1007), and [`riscv64im-unknown-none-elf` staticlib compilation](https://github.com/0xPolygonHermez/zisk/pull/1013).
    - [Gave accelerated operations a dedicated scratch arena separate from the guest heap](https://github.com/0xPolygonHermez/zisk/pull/1024) (ongoing).
    - [Pad to word alignment instead of trimming in ELF section extraction](https://github.com/0xPolygonHermez/zisk/pull/1025).
    - [Guard against addition overflows in the bump allocator](https://github.com/0xPolygonHermez/zisk/pull/1034).
    - [Updated `zisk-eth-client` to bincode 2.0 and convert input blocks](https://github.com/0xPolygonHermez/zisk-eth-client/pull/65).
